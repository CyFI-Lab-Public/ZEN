## 1. Overview
This repository contains the artifact for **ZEN**, an automated memory-forensics pipeline that:

* Recovers a deployed model’s **Mathematical Representation (MR)** and **Programmatic Representation (PR)** from RAM.  
* Synthesises them into a **Complete-Model Representation (CMR)**.  
* Attributes the recovered model to its most-likely open-source base model.  
* Generates a **patch** that lets you re-execute the customised model inside the base model’s environment.

The demo walks through a customised **YOLOv5-Involution** object-detection model trained on VisDrone.
All commands below mirror those used in the paper and in the “Instructions for Demo” document.

## 4. Quick-Start (10-minute demo)

1. **Acquire a memory image** while the deployed model is running. We already have the dump that we are interested in (yolov5-involution), but for other models simply change the name and path to what you want. The dump is available [here](https://drive.google.com/file/d/1nsTN716aGoT7AU32HmuFQjCYYXmakjHR/view?usp=sharing).
```bash
# Load LiME and dump RAM (see LiME README) Skip
sudo insmod lime.ko "path=/path/to/dumps model_dump.lime format=lime"

```
2. **Recover the model's CMR from memory** and perform attribution and patch generation (corresponding to Table 1 in the paper). For the example included with the artifact:
```bash
cd ZEN-AE/volatility3
python vol.py -f ~/ZEN-AE/dumps/yolov5-involution.lime linux.pytorch_cmr_recovery --pid 4009375 --PyVersion 3_8_18 --BaseModelCMR 0 --ModelName yolov5-involution --CompareOnly 0
```
> **Tip:** To add a new model to the BML,  change the --BaseModelCMR 0 to --BaseModelCMR 1


3. **Run the deployed model in a base model environment**

First you can test the original yolov5 model on the test images:
```bash
cd ~/ZEN-AE/Models/YoloV5s/yolov5
python detect.py --weights ~/ZEN-AE/Models/YoloV5s/yolov5/yolov5s.pt
```
You will see that the output images found in "runs/detect/exp" have bounding boxes based on the results of the original model's output (yolov5 trained on COCO)

Next, you can try to run the weights of the Visdrone trained model (utilizing involution) in the regular yolov5 environment to make detections on the example images with the drone model. This is described in Section 2 of the main paper.
```bash
python detect.py \
    --weights ../yolov5-CBAM-Involution-Visdrone/visdrone.pt
```
However, the process will error out saying that it could not find the CBAM layer. This is because the model has not been patched with the pickle file generated by ZEN.

Finally, using the ZEN patch file, we can patch the base environment to fix the errors impeding the use of the visdrone weights (includes patches for CBAM, involution, etc.). This is described in Section 2 and shown in Table 1. Other patching results are shown in Table 2.

```bash
python detect.py --weights ~/ZEN-AE/Models/YoloV5s/yolov5-CBAM-Involution-Visdrone/visdrone.pt --patch ~/ZEN-AE/volatility3/PR/yolov5-involution_3_8_18/pickle_dict_4009375_3_8_18_yolov5-involution.pkl
```

4. **Inspect results** – detections should focus on pedestrians & cars from a drone perspective, matching Fig. 3 in the paper.

*`detect.py` dynamically loads the patch at line 104 via `Patcher.patch_everything_in_filter()`*.
## 5. Full Workflow

| Phase | Command / Action | Explanation                                                                                              |
|-------|------------------|----------------------------------------------------------------------------------------------------------|
| **(A) Deploy model** | `python detect.py …` | Runs the customised YOLOv5-Involution so it is resident in RAM.                                          |
| **(B) Capture RAM** | LiME dump | Dumps live memory to `~/dumps/`                                                                          |
| **(C) Recover CMR** | Volatility plugin above | Produces MR/PR JSONs + pickle patch in `volatility3/PR/…`                                                |
| **(D) Attribute Model** | Automatic inside plugin | Similarity scores & identified base model are printed to stdout (see §IV-B of paper).                    |
| **(E) Patch & Re-execute** | `python detect.py --patch …` | Adds missing layers (e.g., `CBAM`, `Involution`) just-in-time; commenting these lines will crash the model |

## 6. Results & Expected Output
* **Recovered MR:** 9,335,494 weights, 230 tensors, 34 layers (Table 2, paper).  
* **Similarity Scores:** 0.61 (MR SS), 0.97 (PR SS), 0.88 (CMR SS) (Table 3 and 4).  
* **Patch Contents:** 9 new classes, 3 modified fns, 8 added fns (Table 2).  
* **Correct Model Behavior** The model does not crash, and correct images are output in the `yolov5/runs/` folder.
